Complete reference for all API routes, prompt-builder functions, and Supabase query patterns.

## `POST /api/generate`

Generates AI content for a single post using the Anthropic API.

**File:** `app/api/generate/route.ts`
**Runtime:** `nodejs`, `maxDuration: 60`

### Authentication
Requires a valid Supabase session.
```typescript
const { data: { user } } = await supabase.auth.getUser()
if (!user) return NextResponse.json({ error: 'Unauthorised' }, { status: 401 })
```

### Request Body
```typescript
{
  postId: string                  // posts.id
  weekId: string                  // weeks.id
  day: PostDay                    // 'monday'..'saturday'
  pillar: PostPillar              // 'vedic_leadership' | 'banker_coach' | 'coaching_transformation' | 'financial_intelligence' | 'inner_work'
  format: PostFormat              // 'long_form_article' | 'text_post' | 'carousel' | 'market_insights'
  theme: string                   // Weekly theme
  targetAudience: string          // 'Category A' | 'Category B' | 'Category C' | '5-Swans HNI' | 'Bradford'
  targetWordCount: number
  hookIdea?: string | null
  narrativePosition: NarrativePosition  // 'chapter_opening' | 'chapter_deepening' | 'complication' | 'resolution' | 'bridge'
  quarter: string                 // 'Q1'..'Q4'
  marketContext?: string          // Saturday only — real Nifty/Sensex data
  stream?: boolean                // default true
}
```

### Processing Steps
1. Fetch active voice rules: `SELECT * FROM voice_rules WHERE active = true ORDER BY approved_at ASC`
2. Fetch latest story log entry: `SELECT core_insight, thread_planted, references_used FROM story_log ORDER BY created_at DESC LIMIT 1`
3. Fetch week's open thread: `SELECT open_thread FROM weeks WHERE id = $weekId`
4. Build system prompt: `MASTER_SYSTEM_PROMPT + buildVoiceRulesBlock(voiceRules)`
5. Build user prompt:
   - If `format === 'market_insights' && marketContext` → `buildSaturdayMarketInsightsPrompt(...)`
   - Else → `buildNarrativeContext(...)` + `buildGeneratePostPrompt(...)`
6. Call Anthropic (`MODEL`, `MAX_TOKENS`, streaming or non-streaming)
7. Save draft via `saveDrafts()`
8. If long-form (Mon/Wed + `format === 'long_form_article'`) → `generateAndSaveExcerpt()`

### Response — Streaming mode (`stream !== false`)
`Content-Type: text/plain; charset=utf-8`, `Transfer-Encoding: chunked`
Raw text chunks streamed as they're generated by Anthropic. No JSON wrapper.

### Response — Non-streaming mode (`stream === false`)
```typescript
{
  draftId: string
  content: string           // Parsed post content (metadata stripped)
  wordCount: number          // Computed via countWords(), not LLM-reported
  linkedinExcerpt: string | null  // Present only for long-form Mon/Wed
}
```

### Database writes
- `INSERT INTO drafts (post_id, version, content, word_count, is_original)` — twice on first generation (version 1 = original, version 2 = working copy); once on subsequent generations
- `UPDATE posts SET status = 'draft' WHERE id = $postId`
- For long-form: `UPDATE drafts SET linkedin_excerpt = $excerpt WHERE id = $draftId`

## `POST /api/publish`

Publishes, schedules, or previews a post on LinkedIn.

**File:** `app/api/publish/route.ts`
**Runtime:** `nodejs`

### Authentication
Two modes:
- **User-initiated:** Supabase session required
- **Cron-initiated:** `x-cron-secret` header must equal `CRON_SECRET` env var (uses `createServiceClient()`)

```typescript
const isCron = req.headers.get('x-cron-secret') === process.env.CRON_SECRET
const supabase = isCron ? createServiceClient() : createClient()
```

### Request Body
```typescript
{
  postId: string
  publishNow: boolean
  scheduledAt?: string        // ISO 8601 — schedule-only mode
  preview?: boolean           // true => LOGGED_IN visibility (LinkedIn preview)
  promotePreview?: boolean    // convert existing preview post to PUBLIC
  linkedinPostId?: string     // required when promotePreview = true
}
```

### Processing Steps
1. Fetch post + drafts + week:
   ```sql
   SELECT id, day, status, week_id,
          drafts(id, content, linkedin_excerpt, is_original, version),
          weeks(id, week_start)
   FROM posts WHERE id = $postId
   ```
2. Validate status — must be `'approved' | 'scheduled' | 'published'` unless `preview` or `promotePreview`
3. Select current draft (highest version, `is_original = false`; fallback to first draft)
4. Determine `isLongForm = post.day === 'monday' || post.day === 'wednesday'`
5. `resolvePublishText(content, linkedin_excerpt, isLongForm)` → text to publish
6. **If `scheduledAt` and not `publishNow`/`preview`:** update `posts.status = 'scheduled'`, `posts.scheduled_at = scheduledAt`, return early
7. Fetch LinkedIn token (by `user_id` for user calls, `.limit(1).single()` for cron — single-user app)
8. Check token expiry — return 402 if expired
9. **If `promotePreview`:** delete the existing preview LinkedIn post, then publish with `PUBLIC` visibility, `upsert` into `linkedin_posts`
10. **Otherwise:** publish with `preview ? 'LOGGED_IN' : 'PUBLIC'` visibility
11. On success (non-preview): `INSERT INTO linkedin_posts`, `UPDATE posts SET status = 'published'`
12. On failure (non-preview): `UPDATE posts SET status = 'publish_failed'`

### Response
```typescript
{
  published: boolean
  preview: boolean
  url?: string                 // https://www.linkedin.com/feed/update/{urn}/
  linkedinPostId?: string       // e.g. "urn:li:share:123456789"
  wasExcerpt?: boolean          // true if linkedin_excerpt or truncation was used instead of full content
}
```

### `resolvePublishText()` Logic
```
fullContent.length <= 3000 chars
  → return fullContent  (most text posts)

isLongForm AND linkedin_excerpt exists AND linkedin_excerpt.length <= 3000
  → return linkedin_excerpt

else: smart truncation
  → cut fullContent at last "\n\n" before char 2900
  → if no good break found, cut at last "." before char 2900
  → append "\n\n[Full article on coachsharath.com — link in bio]"
```

### LinkedIn API call — `callLinkedInAPI()`
```
POST https://api.linkedin.com/rest/posts
Headers:
  Authorization: Bearer {access_token}
  Content-Type: application/json
  LinkedIn-Version: 202501
  X-Restli-Protocol-Version: 2.0.0

Body:
{
  "author": "urn:li:person:{linkedin_id}",
  "commentary": "{publishText}",
  "visibility": "PUBLIC" | "LOGGED_IN" | "CONNECTIONS",
  "distribution": {
    "feedDistribution": "MAIN_FEED",
    "targetEntities": [],
    "thirdPartyDistributionChannels": []
  },
  "lifecycleState": "PUBLISHED",
  "isReshareDisabledByAuthor": false
}
```
**Post ID extraction:** from `Location` response header (e.g. `urn:li:share:123456789`)

If `linkedin_id` is not stored, falls back to `GET https://api.linkedin.com/v2/userinfo` to fetch `profile.sub`.

### `deleteLinkedInPost()`
```
DELETE https://api.linkedin.com/rest/posts/{linkedinPostId}
Headers:
  Authorization: Bearer {access_token}
  LinkedIn-Version: 202501
  X-Restli-Protocol-Version: 2.0.0
```
Falls back to `DELETE /v2/ugcPosts/{id}` for legacy post IDs created before the migration. Failures are logged but non-fatal (`console.warn`).

## `POST /api/learn`

Called when Sharath approves a post. Extracts story log metadata and candidate voice rules.

**File:** `app/api/learn/route.ts`
**Runtime:** `nodejs`, `maxDuration: 30`

### Authentication
Requires a valid Supabase session.

### Request Body
```typescript
{
  postId: string    // posts.id — the route fetches both drafts from the DB itself
}
```

### Processing Steps
1. Fetch all drafts for `postId` from the DB, ordered by version ASC
2. Identify `original` (`is_original = true`) and `current` (`is_original = false`, latest version)
3. **Always:** call Anthropic with `buildStoryLogExtractionPrompt(currentContent)` → extract `core_insight`, `callback_used`, `thread_planted`, `references_used`
4. **Always:** upsert into `story_log` (one row per post, `onConflict: 'post_id'`)
5. **Always:** if `thread_planted` is set, `UPDATE weeks SET open_thread = $thread_planted WHERE id = $post.week_id`
6. **Always:** `UPDATE posts SET status = 'approved', approved_at = now() WHERE id = $postId`
7. **If content changed:** call Anthropic with `buildEditDiffPrompt(originalContent, currentContent)` → extract 1–5 voice rules as JSON array
8. Return candidate rules for UI review — rules are **not** saved automatically

Candidate rules are saved by the user after reviewing in `CandidateRulesModal` via `POST /api/rules/candidates`.

### Response
```typescript
{
  candidateRules: Array<{
    category: RuleCategory
    rule_text: string
    example_before: string | null
    example_after: string | null
  }>
  storyLog: {
    core_insight: string | null
    callback_used: string | null
    thread_planted: string | null
    references_used: { vedic: string[]; banking: string[]; coaching: string[] }
  } | null
}
```

## `POST /api/plan`

Generates weekly theme proposals and the 6-post week plan.

**File:** `app/api/plan/route.ts`

### Two sub-actions (likely distinguished by an `action` field or two separate endpoints — `propose-themes` and `generate-week`)

#### Theme Proposal
Uses `buildThemeProposalPrompt({ weekNumber, year, quarter, quarterTheme, recentThemes, openThread })`.

**Returns:** JSON array of 5 theme objects:
```json
[
  {
    "theme": "...",
    "rationale": "...",
    "primary_pillar": "vedic_leadership | banker_coach | coaching_transformation | financial_intelligence | inner_work",
    "primary_audience": "Category A | Category B | Category C | 5-Swans HNI | Bradford",
    "open_thread_link": "... | null"
  }
]
```

#### Week Plan Generation
Uses `buildWeekPlanPrompt({ theme, quarter, quarterTheme, weekNumber })`.

**Returns:** JSON array of 6 plan slot objects:
```json
[
  {
    "day": "monday | tuesday | wednesday | thursday | friday | saturday",
    "pillar": "vedic_leadership | banker_coach | coaching_transformation | financial_intelligence | inner_work",
    "format": "long_form_article | text_post | carousel | market_insights",
    "narrative_position": "chapter_opening | chapter_deepening | complication | resolution | bridge",
    "target_audience": "Category A | Category B | Category C | 5-Swans HNI | Bradford",
    "target_word_count": 950,
    "hook_idea": "..."
  }
]
```

**Database writes:** `INSERT INTO posts (week_id, day, pillar, format, narrative_position, target_audience, target_word_count, hook_idea, status)` for each of the 6 slots. Saturday's `status` is set to `'awaiting_market_data'`.

## `POST /api/drafts/save`

Auto-saves editor content. **Updates the current draft in-place** — does not insert a new version.

**File:** `app/api/drafts/save/route.ts`

### Request Body
```typescript
{
  postId: string
  content: string
}
```

### Processing
1. Compute `word_count = countWords(content)` (imported from `lib/anthropic/client`)
2. `UPDATE drafts SET content, word_count WHERE post_id = $postId AND is_original = false`
3. `UPDATE posts SET status = 'edited' WHERE id = $postId AND status = 'draft'`
4. Does **not** trigger `/api/learn` — approval is an explicit user action

A new draft version is only inserted when `/api/generate` runs again (regeneration creates a new `is_original: false` row with an incremented version number).

### Response
```typescript
{ saved: true, wordCount: number }
```

## `GET /api/cron/publish`

Vercel Cron — runs daily at **2:30 AM UTC** (`30 2 * * *`).

**File:** `app/api/cron/publish/route.ts`
**Runtime:** `nodejs`, `maxDuration: 30`, `dynamic: force-dynamic`

### Authentication
`Authorization: Bearer ${CRON_SECRET}` header. Vercel sets this automatically when invoking the route.

### Processing
1. `SELECT id, scheduled_at FROM posts WHERE status = 'scheduled' AND scheduled_at <= now() LIMIT 10`
2. For each matching post, `POST /api/publish { postId, publishNow: true }` with `x-cron-secret` header
3. Returns `{ published: number, failed: number }`

## `GET /api/cron/analytics`

Vercel Cron — runs daily at **1:00 AM UTC** (`0 1 * * *`).

**File:** `app/api/cron/analytics/route.ts`
**Runtime:** `nodejs`, `dynamic: force-dynamic`

### Authentication
`Authorization: Bearer ${CRON_SECRET}` header.

### Processing
1. `SELECT id, linkedin_post_id FROM linkedin_posts WHERE published_at >= now() - 90 days LIMIT 50`
2. Requires `LINKEDIN_SERVICE_TOKEN` env var — if not set, logs a warning and returns `{ fetched: 0 }`
3. For each post, `GET https://api.linkedin.com/v2/socialActions/{linkedin_post_id}` for engagement counts
4. `INSERT INTO performance_data (linkedin_post_id, impressions, likes, comments, shares, clicks, fetched_at)`

## Prompt Builder Functions Reference

All in `lib/anthropic/prompts.ts` unless noted.

| Function | Purpose | Returns |
|----------|---------|---------|
| `MASTER_SYSTEM_PROMPT` | The full Sharath voice/business playbook. Used as base of every system prompt. | `string` constant |
| `buildNarrativeContext(context)` | Builds the "Narrative Context Packet" injected into generation prompts | `string` |
| `buildVoiceRulesBlock(rules)` | Formats active voice rules grouped by category | `string` |
| `buildGeneratePostPrompt(params)` | Full user-turn prompt for a specific post (day/pillar/format/audience-aware) | `string` |
| `buildLinkedInExcerptPrompt(fullArticle)` | Generates a ≤2700-char LinkedIn excerpt from a long-form article | `string` |
| `buildStoryLogExtractionPrompt(postContent)` | Extracts `core_insight`, `callback_used`, `thread_planted`, `references_used` as JSON | `string` |
| `buildEditDiffPrompt(originalDraft, editedDraft)` | Compares drafts, extracts voice rules as JSON array | `string` |
| `buildThemeProposalPrompt(params)` | Generates 5 weekly theme options as JSON | `string` |
| `buildWeekPlanPrompt(params)` | Generates the 6-slot weekly content plan as JSON | `string` |
| `buildSaturdayMarketInsightsPrompt(params)` | (in `saturday-prompt.ts`) Saturday-specific prompt using real market data | `string` |

### `parseGenerationMetadata(rawText)` — `lib/anthropic/client.ts`
Parses the structured output appended to every generated post:
```
{post content}

WORD_COUNT: 1024
CORE_INSIGHT: ...
CALLBACK_USED: ...
THREAD_PLANTED: ...
REFERENCES: {"vedic":[...],"banking":[...],"coaching":[...]}
HASHTAGS: #CoachSharath #5Swans #BradfordInternationalAlliance #Tag4 #Tag5 #Tag6
```
Returns:
```typescript
{
  content: string            // post text with metadata stripped
  wordCount: number          // RECOMPUTED via countWords(content) — LLM value discarded
  coreInsight: string | null
  callbackUsed: string | null
  threadPlanted: string | null
  referencesUsed: { vedic: string[], banking: string[], coaching: string[] }  // note: referencesUsed, not references
  hashtags: string[]
}
```

### `countWords(text)` — `lib/anthropic/client.ts` and `lib/utils/helpers.ts`
Programmatic word counter. Exported from both files: route files import from `lib/anthropic/client`, UI components import from `lib/utils/helpers`. Both implementations are identical.

## Supabase Query Patterns Reference

### Fetch current draft for a post
```typescript
const { data: post } = await supabase
  .from('posts')
  .select(`id, day, status, week_id,
           drafts(id, content, linkedin_excerpt, is_original, version),
           weeks(id, week_start)`)
  .eq('id', postId)
  .single()

const currentDraft = post.drafts
  .filter(d => !d.is_original)
  .sort((a, b) => b.version - a.version)[0]
  ?? post.drafts[0]
```

### Fetch active voice rules
```typescript
const { data: voiceRules } = await supabase
  .from('voice_rules')
  .select('*')
  .eq('active', true)
  .order('approved_at', { ascending: true })
```

### Fetch most recent story log entry
```typescript
const { data: prevStoryLog } = await supabase
  .from('story_log')
  .select('core_insight, thread_planted, references_used')
  .order('created_at', { ascending: false })
  .limit(1)
  .maybeSingle()
```

### Fetch LinkedIn token (single-user pattern)
```typescript
const tokenQuery = userId
  ? supabase.from('linkedin_tokens').select('*').eq('user_id', userId).single()
  : supabase.from('linkedin_tokens').select('*').limit(1).single()
```

## Additional API Routes (not in original docs)

### `GET /api/analytics`
Returns analytics dashboard data. No body.
- Joins `posts → linkedin_posts → performance_data` (latest snapshot per post)
- Returns: `{ hasLinkedInData, totalPublished, totals, trend, byPillar, recentPosts }`
- `trend` = comparison of last 30 days vs prior 30 days (percentage change)
- `byPillar` = per-pillar impression/likes/comments/shares aggregates

### `GET /api/arc`
Returns current year's arc data for the 52-week timeline page.
- Returns: `{ year, currentWeek, openThread, arc: { q1_theme…q4_theme }, weekMap }`
- `weekMap` keys are week numbers 1–52; values include posts with their story_log entries

### `GET /api/calendar?year=&month=`
Returns calendar entries for a given month.
- Returns: `{ year, month, entries: CalendarEntry[] }`
- Each entry: `{ date, weekId, weekTheme, weekNumber, post: { id, day, pillar, format, status, hook_idea, target_word_count } }`

### `GET /api/drafts/[draftId]`
Returns `{ id, content, word_count, version, is_original, created_at }` for a single draft.

### `POST /api/fix-hook`
Rewrites the opening paragraph to fit within 210 characters.
- Body: `{ postId: string, content: string, pillar: string }`
- Uses a targeted Anthropic call (`max_tokens: 256`) to rewrite the hook
- Updates the current draft in-place
- Returns: `{ fixedContent: string }`

### `GET /api/linkedin/status`
Returns: `{ connected: boolean, tokenInfo: { display_name, expires_at, connected_at } | null }`

### `GET /api/linkedin/callback`
LinkedIn OAuth callback. Redirects to `/dashboard/settings?linkedin=connected` on success.

### `DELETE /api/linkedin/disconnect`
Deletes the `linkedin_tokens` row for the current user.
Returns: `{ disconnected: true }`

### `GET /api/posts/[postId]`
Full post data for the draft editor.
Returns: `{ post, originalContent, currentContent, wordCount, hashtags, versions, currentVersionId }`

### `PATCH /api/posts/[postId]`
Safe status transition. Allowed values: `'approved' | 'draft' | 'edited'` only.
Body: `{ status: string }`
Returns: `{ post: { id, status } }`

### `DELETE /api/publish/delete`
Delete a preview post from LinkedIn and reset post to `'approved'`.
Body: `{ postId: string, linkedinPostId: string }`
Note: uses the legacy `/v2/ugcPosts` endpoint for the DELETE call.

### `GET /api/rules`
Returns: `{ rules: VoiceRule[] }` ordered by `created_at DESC`.

### `POST /api/rules`
Body: `{ category, rule_text, example_before?, example_after? }`
Returns: `{ rule: VoiceRule }`

### `PATCH /api/rules`
Body: `{ id: string, active?: boolean, rule_text?: string }`
Returns: `{ rule: VoiceRule }`

### `DELETE /api/rules`
Body: `{ id: string }`
Returns: `{ deleted: true }`

### `POST /api/rules/candidates`
Saves the subset of candidate rules Sharath approved in `CandidateRulesModal` after `/api/learn`.
Body: `{ rules: Array<{ category, rule_text, example_before?, example_after? }>, sourcePostId: string }`
Returns: `{ saved: number }`

### `GET /api/settings`
Returns: `{ settings: SystemSettings, derived: DerivedSettings }`
`derived` includes: `isTrainingPeriod`, `arcWeekNumber`, `arcQuarter`, `arcQuarterTheme`, `arcYearNumber`

### `PATCH /api/settings`
Body: `{ key: string, value: string }`
Returns: `{ updated: true }`

### `POST /api/weeks`
Create or upsert a week. Auto-creates `annual_arcs` row for the year if needed.
Body: `{ weekNumber, year, quarter, theme }`
Returns: `{ weekId: string }`

### `GET /api/weeks/status?weekNumber=&year=`
Returns: `{ week: { weekId, theme, status, approvedCount, totalPosts } | null }`
`totalPosts` excludes Saturday (`awaiting_market_data`) from the denominator.
